## 10. Screenshot capture and annotation plan  
## Take Control of Your Inbox: Outlook on iPadOS  
## 1. Purpose  
Use this plan to capture, annotate, review and maintain every screenshot required for the Outlook on iPadOS guide.  
The screenshot set must:  
* Match the approved 125-page copy deck  
* Reflect the company-issued iPad Pro and managed Outlook environment  
* Use landscape orientation as the primary view  
* Include portrait screenshots only when the interface changes meaningfully  
* Distinguish Outlook controls from iPadOS controls  
* Support touch, Magic Keyboard and trackpad instructions  
* Use fictional, low-risk content  
* Avoid exposing employee, customer, producer or account information  
* Remain readable in Word and PDF  
* Support future image replacement without rebuilding the guide  
  
## 2. Capture environment  
## Primary setup  
Capture using:  
* Company-issued iPad Pro, 5th generation  
* iPadOS 26.5  
* Jamf-managed device  
* Current managed Outlook app  
* Company Microsoft 365 work account  
* Microsoft 365 Copilot license  
* Primary Exchange Online mailbox  
* Approved test shared mailbox  
* Apple Magic Keyboard attached  
* Built-in trackpad  
* Landscape orientation  
* Standard light theme  
* Default or approved text size  
* Stable Wi-Fi connection  
* Dummy mailbox content only

Use portrait orientation only for:  
* Settings screens that display more clearly in portrait  
* Camera or photo workflows  
* Full-screen message views  
* Mobile controls that change substantially  
* Detached-iPad field scenarios

## 9. Dimming and emphasis  
Use subtle dimming when several controls compete for attention.  
Recommended:  
* Light translucent overlay  
* One undimmed area  
* One to three callouts  
Do not combine:  
* Heavy dimming  
* Several large arrows  
* Multiple finger icons  
* Large colored outlines  
* Decorative device frames  
Use a tighter crop when practical.

## Alt-text requirements  
Describe:  
* Application or iPadOS screen  
* Orientation when relevant  
* Highlighted control  
* Purpose of the control  
Do not:  
* Start with “Image of”  
* Repeat every visible label  
* Describe decorative colors  
* Restate the full procedure  
* Depend on the filename

* Lost-device procedures  
* Authentication codes  
Use live text, tables or diagrams instead.  
  
## 15. Placement rules  
Place each screenshot:  
* Immediately after the related step  
* On the same page as the task  
* At a size that preserves text readability  
* After enough context to explain the purpose  
* Before interpretation when the image establishes the screen  
Do not:  
* Separate the screenshot from its procedure  
* Split image and caption across pages  
* Float images without checking reading order  
* Put essential steps only inside the image  
* Reuse one screenshot for unrelated tasks  
* Crop so tightly that the app or account context is unclear

## 19. Technical review checklist  
## Device accuracy  
* Correct iPad model or equivalent layout  
* Correct iPadOS version  
* Correct Outlook version  
* Landscape and portrait labels are accurate  
* Magic Keyboard is represented correctly  
* Jamf-managed controls are not presented as user-controlled  
## Outlook accuracy  
* Correct account is selected  
* Folder and command names are accurate  
* Swipe actions match the configuration  
* Pin and Snooze are validated  
* Shared-mailbox workflow is correct  
* Copilot is shown only in supported contexts  
* Availability-dependent features are labeled  
## Visual quality  
* Image is sharp  
* Interface text is readable  
* Crop retains needed context  
* Maximum three primary callouts  
* Callouts do not overlap controls  
* Bluejay treatment is consistent  
* Orientation is correct  
## Privacy  
* Dummy names only  
* Dummy addresses only  
* No real mailbox content  
* No personal photos  
* No personal Apple ID  
* No verification codes  
* No customer or producer information  
* No device identifiers  
* No confidential links  
## Accessibility  
* Alt text explains the purpose  
* Gesture direction is described in text  
* Color is not the only signal  
* Numbered callouts follow logical order  
* Caption is included  
* Essential instructions remain live text  
* Image is readable at 200% zoom  
* Input method is identified
