## 11. Platform-difference and master-guide integration map  
## Take Control of Your Inbox: Outlook on iPadOS  
## 1. Purpose  
Use this author-facing map to:  
* Keep iPad procedures distinct from Windows and browser workflows  
* Identify content that can be reused across all Outlook guides  
* Document mobile limitations and platform handoffs  
* Separate Outlook, iPadOS and Jamf-managed controls  
* Prepare the future task-first master guide  
* Maintain consistent terminology, content IDs and links  
* Track changing Outlook mobile and Copilot capabilities  
  
## 2. Guide-series structure  
The Outlook guide series includes:  
1. Classic Outlook for Windows  
2. New Outlook for Windows  
3. Outlook on the web  
4. Outlook on iPadOS  
5. Task-first Outlook master guide  
Each detailed guide follows the same major task sequence:  
1. Identify and access the platform  
2. Get oriented  
3. Work with email  
4. Attach, open and share files  
5. Make follow-up visible  
6. Organize the inbox  
7. Search and find  
8. Use calendar follow-up  
9. Configure Outlook  
10. Work with shared mailboxes  
11. Use Copilot  
12. Troubleshoot and get help  
This shared sequence helps employees find the same task after moving between devices.  
  
## 3. Source-of-truth hierarchy  
Validate instructions in this order:  
1. Approved BCBSKS workflows and terminology  
2. Tested behavior on the managed company iPad  
3. Jamf and Microsoft 365 configuration  
4. Current Microsoft and Apple documentation  
5. Existing guide copy  
When documented behavior differs from the managed device:  
* Document what employees currently see  
* Add an organization-managed or availability note  
* Record the difference in author notes  
* Confirm the behavior with the technical owner  
* Do not publish unavailable controls as standard steps  
  
## 4. Platform labels  
Use these labels consistently.  
## Same purpose, different steps  
The task exists across platforms, but controls or navigation differ.  
## Limited on iPad  
The task exists but offers fewer options or less detailed administration.  
## Complete on Windows or web  
The full task is unavailable or impractical in Outlook on iPadOS.  
## Outlook action  
The action occurs inside the Outlook app.  
## iPadOS action  
The action occurs in iPad Settings, Files, the share sheet, the Home Screen or another device control.  
## Organization managed  
Jamf, Microsoft 365 policy or security configuration controls the option.  
## Permission required  
Approved access must be assigned before the task can be completed.  
## Availability may vary  
The feature depends on:  
* Outlook version  
* iPadOS version  
* Microsoft rollout  
* License  
* Mailbox type  
* Permission  
* Jamf configuration  
* Connected experiences  
## Validate before publication  
Use for features that change frequently or have not been confirmed on the issued device.

## 24. Common copy that should remain identical  
Use the same wording across all guides.  
## Core workflow  
**Email → Decide → Make visible → Follow through**  
## Decision questions  
1. Does this need action from me?  
2. What is the next step?  
3. Who owns it?  
4. When does it need to happen?  
5. Where will I make it visible?  
## Four doors  
* Respond now  
* Do the task  
* Make follow-up visible  
* Let it go  
## Follow-up principle  
Visible beats remembered.  
## System principle  
A simple system you use is better than a perfect system you ignore.  
## Copilot principle  
Copilot starts the draft. You finish the message.  
## Shared-mailbox principle  
Confirm ownership, the selected account and the From address before sending.  
## iPad principle  
Use the iPad for timely field work. Use Windows or the web when the task needs more control.

## 32. Final integration decisions  
The following decisions are locked:  
1. Outlook on iPadOS remains a standalone detailed guide.  
2. Landscape mode is the primary Outlook screenshot orientation.  
3. Touch is the default input method.  
4. Magic Keyboard and trackpad alternatives appear only where useful.  
5. Outlook, iPadOS and organization-managed controls are clearly separated.  
6. Common decision guidance remains identical across platform guides.  
7. Exact steps, gestures and screenshots remain platform-specific.  
8. Rules, Sweep, Quick Steps and advanced mailbox administration redirect to Windows or the web.  
9. Shared-mailbox everyday work is included; advanced administration redirects.  
10. Mobile Copilot features must be validated before publication.  
11. Unsupported or managed controls will not be replaced with workarounds.  
12. The future master guide will use a task-first comparison model.  
13. The next production deliverable is the **Word production and publishing plan for Outlook on iPadOS**.
