## Page 28 — Read a message  
## Read a message  
## Use this when  
Open an email, understand the request and decide what should happen next.  
## Open the Inbox

4. Confirm whether you are viewing **Focused** or **Other**.

1. Tap the message in the message list.  
2. Read it in the Reading Pane or full-screen view.  
3. Review:  
    * Sender  
    * To and Cc recipients  
    * Subject  
    * Date and time  
    * Attachments  
    * Links  
    * Requested action  
    * Deadline  
## Ask the key question  
**Does this need action from me?**  
If yes, decide whether to:  
* Reply  
* Reply All  
* Forward  
* Complete the task  
* Flag or Snooze the message  
* Create a task  
* Reserve calendar time

## Page 31 — Use Reply All or Forward  
## Use Reply All  
Use **Reply All** when everyone included needs your response.  
1. Open the message.  
2. Tap the response control.  
3. Select **Reply All**.  
4. Review the To and Cc recipients.  
5. Remove anyone who does not need the response.  
6. Write the message.  
7. Review and send it.  
## Forward a message  
Use **Forward** when a new recipient needs the original context.  
1. Open the message.  
2. Tap the response control.  
3. Select **Forward**.  
4. Add the new recipient.  
5. Add a short explanation above the forwarded message.  
6. Review included attachments.  
7. Remove unnecessary information when appropriate.  
8. Review and send.  
## Add useful context  
Example:  
Please review the request below and confirm whether you can provide the missing effective date by Thursday.  
## Decision guide  

| Command   | Use when                             |
| --------- | ------------------------------------ |
| Reply     | Only the sender needs the response   |
| Reply All | Everyone included needs the response |
| Forward   | A new recipient needs the context    |

## Page 32 — Create a new email  
## Create a new email  
## Use this when  
Start a new topic that does not belong in an existing conversation.  
## Start the message  
1. Tap **Compose** or the new-message control.  
2. Enter the primary recipients in **To**.  
3. Add **Cc** or **Bcc** when needed.  
4. Enter a clear subject.  
5. Tap the message body.  
6. Write the email.  
## Use recipient fields correctly  

| Field | Use |
| ----- | -------------------------------------------------------------------------------- |
| To | People expected to act or respond |
| Cc | People who need the information |
| Bcc | People who should receive the message without displaying their address to others |
  
****Write a useful subject****  
## Less useful  
Follow-up  
## More useful  
Action needed: Confirm July 1 effective date by Friday  
## Structure the message  
Use this order:  
1. Purpose  
2. Requested action  
3. Deadline  
4. Supporting details  
5. Closing  
## Example  
**Subject:** Action needed: Confirm renewal effective date by June 20  
Hi Jordan,  
Please confirm the effective date for the renewal request below by June 20.  
We currently have July 1 listed, but the producer’s message references August 1. Once the date is confirmed, I can complete the update.  
Thank you, [Name]

## Page 34 — Save, reopen or discard drafts  
## Save a draft  
Outlook normally saves unfinished messages automatically.  
To return to a draft:  
1. Begin a new message or reply.  
2. Wait briefly for Outlook to save the draft.  
3. Close the message.  
4. Open the account and folder menu.  
5. Tap **Drafts**.  
6. Open the draft.  
7. Continue editing.  
## Confirm whether the message was sent  
Check:  
* **Sent** for delivered messages  
* **Drafts** for unfinished messages  
* A scheduled-message location for messages set to send later  
## Discard a draft  
1. Open the draft.  
2. Tap the delete or discard control.  
3. Confirm the action when prompted.  
Or:  
1. Open **Drafts**.  
2. Select the unwanted draft.  
3. Delete it.  
## Track important drafts  
If the message must be finished later:  
* Add a reminder  
* Create a task  
* Add calendar time  
* Keep the original source email flagged  
## Connectivity note  
When working offline or with a weak connection:  
* Confirm the draft saved.  
* Do not assume the message was sent.  
* Recheck Sent when the connection returns.  
## What to remember  
A draft is unfinished work. Return to it, track it or remove it.  
## Avoid this  
* Do not leave important drafts without a follow-up plan.  
* Do not assume closing the composer sends the message.  
* Do not force-close Outlook before confirming that the draft saved.  
* Do not keep several outdated versions of the same response.

4. Confirm the scheduled time.

2. Select the edit or schedule control.  
3. Change the date and time.  
Or:  
1. Cancel scheduled delivery.  
2. Edit the message.  
3. Send or schedule it again.  
## Check the time zone  
Before scheduling:  
* Confirm the current time zone.  
* Confirm the recipient’s expected business time.  
* Check whether the iPad recently changed locations.  
* Leave enough time before the deadline.

## Page 37 — Understand mobile sending limitations  
## Some sending controls differ on iPad  
Outlook on iPadOS does not always provide the same message controls as Windows or Outlook on the web.  
## Undo Send  
Undo Send may:  
* Be unavailable  
* Use a different workflow  
* Provide only a brief delay  
* Depend on the current app version  
Review the message before sending rather than relying on Undo Send.  
## Message Recall  
Recall is not a dependable mobile recovery method.  
Even when Recall is available in another Outlook platform, success depends on specific mailbox and recipient conditions.  
Use Windows or Outlook on the web when an approved Recall workflow is required.  
## Read and delivery receipts  
Receipt controls may:  
* Be unavailable in the mobile composer  
* Depend on account settings  
* Be blocked by the recipient or organization  
* Fail to confirm understanding or action  
Request a direct reply when confirmation matters.  
## Advanced message options  
Complete on Windows or the web when you need:  
* Advanced delivery controls  
* Detailed receipt settings  
* Complex formatting  
* Message Recall  
* Large recipient management  
* Advanced sensitivity or permission options  
* Detailed add-in controls  
## Mobile decision table  

| Need                  | Recommended action                      |
| --------------------- | --------------------------------------- |
| Stop before sending   | Complete a full review                  |
| Send later            | Schedule Send, where available          |
| Confirm receipt       | Request a direct response               |
| Recall a sent message | Use an approved Windows or web workflow |
| Use advanced options  | Move to Windows or the web              |
  
****What to remember****  
Mobile Outlook is designed for timely field work. Use Windows or the web when the message needs more control.  
## Avoid this  
* Do not depend on Recall to fix an error.  
* Do not treat a read receipt as proof of understanding.  
* Do not send sensitive or complex messages without a complete review.  
* Do not assume every Windows message option exists on iPad.
