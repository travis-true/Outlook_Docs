The guide will preserve the session’s established approach: decide what an email needs, make follow-up visible, use low-risk content with Copilot and review generated output before acting.  
  
## 2. Purpose  
This guide helps sales employees complete common Outlook tasks from a company-issued iPad while working in the field or away from their Windows laptop.  
It combines:  
* Managed-device guidance  
* Touch, keyboard and trackpad instructions  
* Detailed email procedures  
* Mobile file-handling workflows  
* Follow-up decision support  
* Shared mailbox guidance  
* Copilot procedures  
* Platform limitations  
* Safe troubleshooting  
* Clear handoffs to Windows or Outlook on the web  
Employees should be able to locate one task, follow the steps and return to work without reading the entire guide.  
  
## 3. Performance outcomes  
After using the guide, employees should be able to:  
1. Open Outlook on the managed iPad and confirm the correct work account.  
2. Recognize Outlook on iPadOS and distinguish it from Apple Mail or Outlook on the web.  
3. Navigate Outlook using touch, Magic Keyboard and trackpad controls.  
4. Read, create, reply to and forward email.  
5. Attach, open and share approved files.  
6. Decide what an email needs.  
7. Make follow-up visible with flags, Snooze, categories, To Do and calendar cues.  
8. Organize and search email using mobile-supported controls.  
9. Configure employee-controlled Outlook and iPadOS settings.  
10. Add and use an approved shared mailbox.  
11. Use supported Copilot features.  
12. Review Copilot output before sending or acting.  
13. Identify tasks that must be completed on Windows or the web.  
14. Resolve common app, account, file, keyboard and notification issues.  
15. Use the correct support path when settings are managed or unavailable.  
  
## 4. Core workflow  
Use the same framework throughout the guide:  
## Email → Decide → Make visible → Follow through  
Repeat these questions:  
1. **Does this need action from me?**  
2. **What is the next step?**  
3. **Who owns it?**  
4. **When does it need to happen?**  
5. **Where will I make it visible?**  
Use the four-door model:  

| Door | Use when | Action |
| ---------------------- | ---------------------------------- | ------------------------------------------------------- |
| Respond now | A quick reply resolves the request | Reply and send |
| Do the task | The work can be completed quickly | Complete it, then file the message |
| Make follow-up visible | More work or waiting is required | Flag, Snooze, categorize, create a task or reserve time |
| Let it go | No action is required | File, archive or delete |
  
The participant workbook establishes the same action-first workflow and the principle that visible follow-up is more reliable than memory.

## 7. Follow-up tool framework  

| Tool              | Best use on iPad                                        |
| ----------------- | ------------------------------------------------------- |
| Flag              | Track unfinished email                                  |
| Snooze            | Bring a message back at a more useful time              |
| Category          | Identify status, work type or ownership where supported |
| Swipe action      | Complete a common action quickly                        |
| Microsoft To Do   | Track due dates, reminders, notes and steps             |
| My Day            | Focus on selected work today                            |
| Calendar event    | Reserve focused work time                               |
| Meeting           | Coordinate discussion or a decision                     |
| Folder or Archive | Store completed or reference information                |
  
****Pin treatment****  
Include Pin only after validation.  
If Pin is unavailable or inconsistent:  
**Complete on Windows or web:** Use Pin in New Outlook or Outlook on the web. On iPad, use Flag, Snooze or To Do instead.  
## Decision rules  
* Use a **flag** when the email remains unfinished.  
* Use **Snooze** when timing determines when the email becomes useful.  
* Use a **category** to identify status or work type.  
* Use **To Do** for multi-step work or timed reminders.  
* Use a **calendar event** when focused time is needed.  
* Use a **swipe action** only for actions you understand and use consistently.  
* Use the fewest tools necessary.

## Permission required

## Availability may vary  
The control depends on app version, rollout, account, license or configuration.  
## Complete on Windows or web  
The full task is not available or practical on iPad.  
This framework should appear in the legend and throughout configuration, shared mailbox and troubleshooting sections.
